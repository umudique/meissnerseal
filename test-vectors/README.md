# MeissnerSeal Test Vectors

Cryptographic test vectors for MeissnerSeal. Every vector file must be
independently verifiable — either generated from scratch by `cross_verify.py`
using standard Python libraries, or sourced from an authoritative external
authority (NIST ACVP) and confirmed by `mlkem_cross_verify.py`.

## File Formats

### Self-Generated Format

```json
{
  "profile": "KDF_ARGON2ID_V1",
  "version": 1,
  "description": "Human-readable description of what is tested",
  "generated_by": "cross_verify.py (library names)",
  "cases": [
    {
      "id": "descriptive-case-id",
      "inputs": {},
      "expected": {},
      "notes": "optional"
    }
  ]
}
```

### NIST-Sourced Format

```json
{
  "schema": "<name>-<version>",
  "description": "Algorithm, format, and field descriptions",
  "source": {
    "name": "NIST ACVP-Server",
    "repository": "https://github.com/usnistgov/ACVP-Server",
    "file": "path/to/internalProjection.json",
    "commit": "<sha>",
    "algorithm": "ML-KEM-768",
    "testType": "AFT",
    "tcIds": [26, 27, 28]
  },
  "vectors": [
    { "tcId": 26, "ek": "...", "dk": "...", "c": "...", "k": "...", "m": "..." }
  ]
}
```

All byte fields are uppercase hex strings. Lengths for ML-KEM-768:
`ek` = 1184 B, `dk` = 2400 B (FIPS 203 §5.3 expanded), `c` = 1088 B,
`k` = 32 B, `m` = 32 B.

## Cross-Verification Scripts

### `cross_verify.py`

Independently generates and verifies self-generated vectors using Python
cryptography libraries. Does **not** read existing JSON files — computes
from scratch for maximum independence.

```
pip install -r requirements.txt
python3 cross_verify.py           # run all verifications
python3 cross_verify.py kdf       # KDF vectors only
python3 cross_verify.py aead      # AEAD vectors only
```

Used as a CI drift guard: regenerates vectors and fails if `git diff` shows
any change (`test-vectors` job in `ci-fast.yml`).

### `mlkem_cross_verify.py`

Re-fetches `ML-KEM-encapDecap-FIPS203/internalProjection.json` from the NIST
ACVP-Server GitHub repository and confirms that `mlkem_768_kat_v1.json` matches
the upstream source byte-for-byte. Uses stdlib only (no pip install required).

```
python3 mlkem_cross_verify.py          # fetch NIST source and compare
python3 mlkem_cross_verify.py --offline  # format check only (no network)
```

## Protocol

`security_engineering_protocol.md §STEP 3 Path A` governs how vector files
relate to implementation:

- **Self-generated:** `cross_verify.py` must produce the vector file before
  the Rust implementation is written. The implementation is correct when it
  reproduces every value.
- **NIST-sourced:** The NIST ACVP file is the authority. The implementation
  is correct when `mlkem_cross_verify.py` passes and the Rust KAT test passes.

Rust consumers live alongside the implementation under test (`#[cfg(test)]`)
and use `include_str!` to load the JSON at compile time.
